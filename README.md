# CMSC 731 Project — VR Gaussian Splatting with Runtime Variant Switching

A Unity VR application for viewing 3D Gaussian Splatting scenes on Meta Quest, with the ability to switch in-headset between an original captured scene and several processed variants: **style transfer**, **scene/object editing**, and **object removal**.

Built on top of [ninjamode/Unity-VR-Gaussian-Splatting](https://github.com/ninjamode/Unity-VR-Gaussian-Splatting), a VR-capable fork of [aras-p/UnityGaussianSplatting](https://github.com/aras-p/UnityGaussianSplatting).

## Overview

The base project renders a single Gaussian Splat asset in VR. This project extends it with a **Splat Switcher** that lets the user cycle through multiple Gaussian Splat assets at runtime using a controller button, so different processed versions of the same scene can be compared directly in the headset without leaving VR.

The variants explored in this project are:

- **Original** — the unmodified captured Gaussian Splat scene
- **Style transfer** — the scene re-rendered in different artistic styles
- **Editing** — scenes with geometric or appearance edits applied
- **Object removal** — scenes with selected objects removed / inpainted

## Project Structure

The Unity project root is the `VR-URP/` directory.

```
VR-URP/
  Assets/
    GaussianAssets_styles/      # style-transferred Gaussian Splat assets
    GaussianAssets_editing/     # edited Gaussian Splat assets
    GaussianAssets_removal/     # object-removal Gaussian Splat assets
    GS Assets/                  # modified Gaussian Splatting package source
    Scripts/                    # project scripts (incl. SplatSwitcher, VRVerticalMove)
    Prefabs/, Materials/, Settings/
    MRTK.Generated/, XR/, XRI/  # XR / MRTK configuration
    TextMesh Pro/, Samples/
  Packages/                     # Unity package manifest + dependencies
  ProjectSettings/              # Unity project settings
docs/                           # project documentation / report assets
```

`Library/`, `Logs/`, `Temp/`, and `UserSettings/` are intentionally not tracked — they are regenerated by Unity automatically.

## Requirements

- Unity **2022.3.x** (developed on 2022.3.51f1)
- Universal Render Pipeline (URP)
- A platform supporting compute shaders (D3D12 / Vulkan / Metal in editor; Android for the headset build)
- Meta Quest headset (developed and tested on Quest Pro / Quest 3)
- MRTK3 (already included under `Assets/`)

## Setup

1. Clone the repository.
2. If the Gaussian Splat assets are stored via Git LFS, run `git lfs pull` to fetch them. Otherwise see **Gaussian Splat Assets** below.
3. Open the `VR-URP/` folder as a project in Unity Hub (Unity 2022.3.x).
4. Open the main scene: the scene named **Demo** under `Assets/`.
5. Set the build target to **Android** (`File > Build Settings > Android`).

## Gaussian Splat Assets

The Gaussian Splat data files are large. Depending on how this repository was distributed:

- If included via Git LFS, they will download with `git lfs pull`.
- If not included, they must be placed into the corresponding folders:
  - `Assets/GaussianAssets_styles/`
  - `Assets/GaussianAssets_editing/`
  - `Assets/GaussianAssets_removal/`

Each Gaussian Splat asset is created from a source `.ply` via Unity's
`Tools > Gaussian Splats > Create GaussianSplatAsset` menu. The raw `.ply`
inputs are not tracked in this repository.

> **Note:** Use the *High* or *Very High* quality preset when creating assets.
> Lower presets are lossy and can produce visual artifacts when switching.

## How to Use (in VR)

1. Build and deploy to the Quest (`File > Build Settings > Build And Run`, with the headset connected and Developer Mode enabled).
2. Put on the headset. The first Gaussian Splat variant loads automatically and its name is shown on a floating label in front of you.
3. **Press the B button on the right controller** to cycle to the next variant.
4. The displayed splat swaps and the label updates to the new variant's name.
5. The label repositions in front of you on each switch so it is always visible.

Locomotion uses the standard XRI providers (move, turn, teleport). A custom
vertical-movement script (`VRVerticalMove`) allows moving up and down.

### Desktop testing

In the Unity editor, with the Game view focused, the **Left / Right arrow keys**
cycle variants without needing the headset.

## Key Components

### `SplatSwitcher.cs`
`Assets/Scripts/SplatSwitcher.cs`

The core of this project. Holds a list of named Gaussian Splat assets and swaps
the active asset on a target `GaussianSplatRenderer` at runtime. Responsibilities:

- Stores variants as `(displayName, GaussianSplatAsset)` entries.
- On switch, assigns the new asset to the renderer; the renderer's own update
  loop detects the change and rebuilds its GPU resources.
- Reads the right Quest controller's B button (`secondaryButton`) to cycle.
- Applies a configurable cooldown so a single press does not rapid-fire.
- Drives a world-space TextMeshPro label that displays the current variant
  name and repositions in front of the camera on each switch.

Inspector fields:

- **Splat Renderer** — the `GaussianSplatRenderer` whose asset is swapped
- **Splats** — the ordered list of variants (display name + asset)
- **Label / Label Container** — the TMP text and its world-space canvas
- **Label Distance / Label Vertical Offset** — placement relative to the user
- **Swap Cooldown** — minimum seconds between switches

### Modified `GaussianSplatRenderer.cs`
Located in the vendored Gaussian Splatting package source.

Modified from the upstream `ninjamode` fork to:

- Expose / auto-assign the center-eye camera (`m_centerEyeCamera`) so it does
  not need manual Inspector wiring and does not throw when unset in VR.
- Rebuild GPU resources only when the splat asset actually changes, rather than
  also rebuilding whenever the camera reference is momentarily null. This fixes
  per-frame flicker and locomotion stutter that occurred when switching assets.

### `VRVerticalMove.cs`
`Assets/Scripts/VRVerticalMove.cs`

Adds vertical (up/down) movement to the XR Origin via an input action, with a
deadzone to prevent thumbstick drift.

## Known Limitations

- Gaussian Splatting is fill-rate heavy. Standing very close to a dense splat
  scene can drop the Quest framerate significantly (a known limitation of the
  underlying renderer, see
  [aras-p/UnityGaussianSplatting#145](https://github.com/aras-p/UnityGaussianSplatting/issues/145)).
  Mitigations: lower SH order, reduce splat scale, lower the render scale,
  or use lower-density assets.
- Switching very large assets causes a brief one-time hitch while GPU buffers
  are rebuilt.
- The Microsoft MR OpenXR plugin logs a harmless `DllNotFoundException` when the
  project is opened in the macOS editor (the plugin is Windows-only). It does
  not affect the Android/Quest build.
